<objective>
Update broader terminology in templates from GSD-style ("Phase", "Milestone") to GRD research terminology ("Study", "Version").

Purpose: Per CONTEXT.md decisions, "phase" becomes "study" (individual research unit) and "milestone" becomes "version" (release grouping). Templates must reflect this so all newly generated planning files use consistent terminology.

Output: Templates and references updated with study-centric terminology.
</objective>

**Terminology mapping (from CONTEXT.md):**
| Old | New | Context |
|-----|-----|---------|
| "Phase" | "Study" | Individual research unit (e.g., "Phase 11" -> "Study 11") |
| "phase" | "study" | Lowercase references |
| "Milestone" | "Version" | Release groupings (e.g., "v1.0 Milestone" -> "v1.0 Version") |
| "milestone" | "version" | Lowercase references |
| "Milestone History" | "Version History" | STATE.md section header |

**Scope boundaries:**
- Directory paths stay unchanged (`.planning/phases/` not renamed)
- The word "phase" in compound technical terms preserved (e.g., "multi-phase", "out-of-phase")
- Only content terminology changes, not file paths
</context>
